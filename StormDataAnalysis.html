Population Health Impact and Economic Consequences of Storms

Ben Kesseler

May 24, 2016

Synopsis

This report examines the U.S. National Oceanic and Atmospheric Administration’s (NOAA) storm database, comparing economic damages, injuries, and fatalities. All analysis was performed at the national level, ignoring specific locations. Individual “events” are not easily determined to assess impacts per “event”, so impacts are considered in aggregate over time.

The most damaging and dangerous event types are tornados, hurricanes/typhoons, thunderstorm winds, lightning, hail, floods, storm surges/tides, rip currents, and heat.

Heat (combining heat and excessive heat) is the most dangerous event in terms of fatalities, and second most dangerous in terms of injuries, while causing little economic damage, so it should be prepared for in warm climates.

Tornados are very damaging in both health and economic terms, and should be prepared for in areas prone to experience them.

Floods are extremely damaging economically, while also impacting health fairly severely, so flood prone regions should take relevant precautions.

Hurricanes and typhoons, perhaps because of the advanced warning usually available, cause less impact to health, while causing the second highest economic damages.

Initial Cleaning

Each row is a very detailed description of a location, event type, injuries, fatalities, and damage, if any. However, actual logical events such as a regional storm or other weather impact might entail many rows, and there is no obvious way to usefully aggregate the rows to logical events.

The analysis covers the entire United States, so location is not important.

As a result, the data will be aggregated by event type at the year level. This will allow the analysis to calculate the impact at an annual level if needed, and across the entire time period for the main analysis.

Once the data is loaded from the .csv.bz2 file, I keep only the year, event type, fatalities, injuries, property damage, and crop damage.

I convert the economic impacts into a consistent scale. I’m unsure of the meaning of non-standard exponents in the data, so I’m treating them as 0, after documenting their prevalence.

I will also subset the data to any rows with non-0 injuries, fatalities, or economic damage.

Now I’m going to clean up some of the obvious event_types errors.

adjust_events <- function(f_event_type) {
  # Check if correct
  if (check_match(f_event_type)) 
    return(f_event_type) 
  
  # Marine Thunderstorm Wind
  if (grepl("(?=marine)+(?=.*tstm)", f_event_type, perl = TRUE))
    return("marine thunderstorm wind")
  
  # Coastal Flood
  if (grepl("^(?=(coastal|erosion|tidal))+(?=.*flood)+", f_event_type, perl = TRUE)) 
    return("coastal flood")
  
  # Cold Wind Chill
  if (grepl("^(cold|extended)", f_event_type)) 
    return("cold wind chill")
  
  # Dense Fog
  if (grepl("(?!freezing)(?=.*fog)+", f_event_type, perl = TRUE)) 
    return("dense fog")
  
  # Drought
  if (grepl("^(drought)", f_event_type)) 
    return("drought")
  
  # Dust Devil
  if (grepl("^(dust devil)", f_event_type)) 
    return("dust devil")
  
  # Extreme Cold Wind Chill
  if (grepl("^(?=(extreme|record))(?=.*(cold|wind))", f_event_type, perl = TRUE)) 
    return("extreme cold wind chill")
  
  # Flash Flood
  if (grepl("(flash)", f_event_type)) 
    return("flash flood")
  
  if (grepl("(rapidly)", f_event_type)) return("flash flood")

  # Thunderstorm
  if (grepl("^(?!non)(?=.*(thunderstorm|tstm|thune|tund|rtor|deer|restor))+", f_event_type, perl = TRUE))
    return("thunderstorm wind")
  
  # Lakeshore Flood
  if (grepl("(?=lake)+(?=.*flood)", f_event_type, perl = TRUE))
    return("lakeshore flood")
  
  # Frost Freeze
  if (grepl("(frost|freeze)", f_event_type))
    return("frost freeze")
  
  # Hail
  if (grepl("^(?!(gusty))(?=.*hail)+", f_event_type, perl = TRUE))
    return("hail")

  # Excessive Heat
  if (grepl("(?=record)(?=.*heat)", f_event_type, perl = TRUE))
    return("excessive heat")
  
  # Heat
  if (grepl("(heat|warm)", f_event_type))
    return("heat")

  # High Wind
  if (grepl("^(?=(high|gusty))(?=.*wind)", f_event_type, perl = TRUE))
    return("high wind")
  
  # Lightning
  if (grepl("(lightn|lignt)", f_event_type))
    return("lightning")
  
  # Heavy Rain
  if (grepl("(?=(heavy|hvy|exc|torr))(?=.*rain)", f_event_type, perl = TRUE))
    return("heavy rain")
  
  # Heavy Snow
  if (grepl("^(snow|heavy sn|excessive sn)", f_event_type))
    return("heavy snow")
  
  # Rip Current
  if (grepl("^(rip)", f_event_type))
    return("rip current")
  
  # High Surf
  if (grepl("(surf)$", f_event_type))
    return("high surf")
  
  if (grepl("^(heavy surf)", f_event_type))
    return("high surf")
  
  # High Swells
  if (grepl("swell", f_event_type))
    return("high swells")
  
  # Hurricane Typhoon
  if (grepl("(hurricane|typhoon)", f_event_type))
    return("hurricane typhoon")
  
  # Storm Surge Tide
  if (grepl("surge", f_event_type))
    return("storm surge tide")
  
  # Strong Wind
  if (grepl("(?=(strong|(storm force)))+(?=.*wind)+", f_event_type, perl = TRUE))
    return("strong wind")
  
  # High Swells
  if (grepl("(?=(waves| seas))+(?!.*snow)", f_event_type, perl = TRUE))
    return("high swells")
  
  # Tornado
  if (grepl("^(tornado|landspout)", f_event_type))
    return("tornado")
  
  # Tropical Storm
  if (grepl("^tropical", f_event_type))
    return("tropical storm")
  
  # Water Spout
  if (grepl("(spout)", f_event_type))
    return("water spout")
  
  # Wildfire
  if (grepl("fire", f_event_type))
    return("wildfire")
  
  # Winter Storm
  if (grepl("(?=winter)+(?=.*storm)+", f_event_type, perl = TRUE))
    return("winter storm")
  
  # Winter Weather
  if (grepl("^(winter|wintry)", f_event_type))
    return("winter weather")
  
  # Flood
  if (grepl("(flood|fld)", f_event_type))
    return("flood")
}

Fatality, Injury, and Economic Damage Sources

I need to examine the distribution of injuries, fatalaties, and economic damage by the event types present in this data.

I’ll be using top 20 event types (based on the variables being analyzed) for these analyses, to make things easier to work with.

Note, all of the preparation for these plots occurred above, in the data processing section, but the plots are presented and discussed here.

Figure 1 - Injuries & Fatalities Exploration

Figure_1

This plot shows fatalities vs injuries for the top 20 events by combined health impact. Points and labels are colored by the ratio of fatalities to injuries.

Excessive heat, heat, tornados, flash floods, and lightning are the most deadly event types, while tornados, floods, excessive heat, thunderstorm winds, and lightning are the most injurious event types.

Fatalities and injuries seem to be correlated, although Tornados cause signifigantly more injuries than any other event. However, Rip Currents are more dangerous to life, independent of overall risk of encountering the events.

Figure 2 - Economic Damage Exploration

Figure_2

This plot shows economic damage in billions of USD, for the top 20 event types by economic damage. Bars are colored by the economic damage. The values have not been adjusted for inflation.

Floods, hurricanes/typhoons, storm surges/tides, and tornados are the most damaging events, economically.

Flood may be somewhat overstated as all flood events are required to have a damage statement, even if an estimate.

Figure 3 - Economic Damage vs Combined Health Impact

Figure_3

This plot shows economic damage vs combined health impact (injuries + fatalities), both on a log10 scale, for the top 20 event types by combined health impact. Points and labels are colored by the ratio of fatalities to injuries, before log10 conversion.

Event types are fairly evenly distributed on a log scale, in terms of both economic damage and combined health impact.

Rip Current still stands out as a very dangerous event, with extremely small prevalence and near-zero economic impact.

Further Exploration

Location Analysis

A further analysis could plot injuries, fatalities, and economic damage as a density plot over the United States, by utilizing the latitude and longitude present in the data.

Time Analysis

After subsetting to post-1994 data to ensure consistent reporting methods, events and damages could be compared over time, to determine if any events are occuring more frequently, or are causing more impact, over time.

Logical Event Combinations

Some events are logically connected, such as “heat” and “excessive heat”, and might be combined to create a smaller set with more value for communication to the public, as the event types are currently defined for the meteorological community.
